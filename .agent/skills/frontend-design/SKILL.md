# Frontend Design

## Overview
This skill creates distinctive, production-grade frontend interfaces that avoid generic AI aesthetics. It emphasizes quality, consistency, and attention to detail that makes interfaces feel professional and intentional.

## Core Principles

### Distinctive Over Generic
- Avoid "AI slop" - the generic look that screams "generated by AI"
- Create unique visual identity
- Use thoughtful typography, spacing, and color
- Avoid default component libraries without customization

### Production Quality
- Every detail matters
- Consistent spacing and sizing
- Proper accessibility
- Responsive design
- Smooth animations and transitions

### User-Centered Design
- Prioritize usability over aesthetics
- Clear visual hierarchy
- Intuitive interactions
- Feedback for all actions
- Error prevention and recovery

## Design Process

### 1. Understand the Context
- What is the purpose of this interface?
- Who are the users?
- What device(s) will it be used on?
- What is the existing design system (if any)?

### 2. Define Structure
- Layout and composition
- Content hierarchy
- Component breakdown
- Responsive breakpoints

### 3. Style with Intent
- Choose typography deliberately
- Use color meaningfully (not just for decoration)
- Apply spacing consistently
- Add motion purposefully

### 4. Implement and Refine
- Write semantic HTML
- Style with CSS/Tailwind/styled-components
- Add interactions thoughtfully
- Test across devices and states

## Best Practices

### Typography
- Use 2-3 font families maximum
- Establish clear type scale
- Ensure readability (line-height, line-length)
- Use font weights intentionally

### Color
- Limit palette to 5-7 colors
- Use color for meaning, not decoration
- Ensure sufficient contrast
- Support dark mode where appropriate

### Spacing
- Use consistent spacing scale (4px, 8px, 16px, etc.)
- Create visual breathing room
- Group related elements
- Use white space strategically

### Components
- Make components reusable
- Define clear states (default, hover, focus, disabled)
- Handle loading and error states
- Add micro-interactions for delight

### Accessibility
- Use semantic HTML
- Add proper ARIA labels
- Ensure keyboard navigation
- Test with screen readers
- Meet WCAG guidelines

## Anti-Patterns to Avoid

- Using default blue buttons everywhere
- Inconsistent spacing
- Generic placeholder content
- No hover/focus states
- Ignoring mobile layouts
- Over-animating everything
- Using multiple font families
- Cluttered interfaces

## Output Quality

When creating frontend code:
- Clean, well-organized code
- Proper component structure
- Responsive by default
- Accessible from the start
- Styled with consistent system
