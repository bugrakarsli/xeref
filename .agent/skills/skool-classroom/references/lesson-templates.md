# Lesson Templates

Four lesson types, each optimized for a different learning goal. Pick the right type based on what the student needs to learn.

---

## Type 1: Concept Lesson

**When to use**: Teaching a new idea, framework, or mental model before hands-on work.

```markdown
# [Concept Name] — Understanding [Topic]

## Why This Matters
[1-2 paragraphs: real-world problem this concept solves]

## The Core Idea
[Explain the concept in plain language — no jargon]
[Use an analogy the student already understands]

## How It Works
### [Component 1]
[Explanation + visual/diagram description]

### [Component 2]
[Explanation + visual/diagram description]

### [Component 3]
[Explanation + visual/diagram description]

## Real-World Examples
1. **[Example 1]**: [How this concept applies]
2. **[Example 2]**: [How this concept applies]
3. **[Example 3]**: [How this concept applies]

## Common Misconceptions
- ❌ [Wrong belief] → ✅ [Correct understanding]
- ❌ [Wrong belief] → ✅ [Correct understanding]

## Key Takeaway
[One sentence the student should remember]

## Check Your Understanding
- [ ] Can you explain [concept] to someone in 30 seconds?
- [ ] Can you identify where [concept] applies in your own work?
```

**Video length**: 5-8 min  
**Best for**: Module 2 (AI Foundations), Module 7 (API basics), Module 8 (Agent patterns)

---

## Type 2: Tutorial Lesson

**When to use**: Step-by-step walkthrough where the student follows along and builds something.

```markdown
# [Action Verb] [Specific Thing] — Step-by-Step Tutorial

## What We're Building
[Screenshot or description of the end result]
[Why this specific build matters]

## What You'll Need
- [ ] [Tool/account 1] — [link to sign up]
- [ ] [Tool/account 2] — [link to sign up]
- [ ] [Prerequisite knowledge from Module X, Lesson Y]

## Step 1: [Action]
[SCREEN: Show the interface/tool]

1. [Do this specific thing]
2. [Then do this]

**Expected result**: [What the student should see]

⚠️ **Common mistake**: [What often goes wrong + fix]

## Step 2: [Action]
[Continue pattern...]

## Step 3: [Action]
[Continue pattern...]

## Step N: [Final Action]

## Test It
1. [How to verify it works]
2. [What success looks like]

## Troubleshooting
| Problem | Cause | Fix |
|---------|-------|-----|
| [Issue 1] | [Why] | [How to fix] |
| [Issue 2] | [Why] | [How to fix] |
| [Issue 3] | [Why] | [How to fix] |

## What You Built
[Recap of the complete deliverable]

## Next Step
[What to try next — link to next lesson or challenge]
```

**Video length**: 10-15 min  
**Best for**: Module 4 (First Automation), Module 9 (CRM), Module 10 (Marketing), Module 14 (Claude Code)

## Type 3: Workshop Lesson

**When to use**: Hands-on build session where the student creates something unique (not just following steps).

```markdown
# Workshop: Build Your Own [Thing]

## The Challenge
[Describe what the student will create — leave room for creativity]

## Constraints
- Must use: [specific tool or technique]
- Must include: [required feature]
- Time limit: [suggested time]
- Level: [complexity level 1-4]

## Starter Template
[Provide a starting point — JSON, code snippet, or workflow export]

## Building Blocks
### Option A: [Approach 1]
[When to choose this + quick how-to]

### Option B: [Approach 2]
[When to choose this + quick how-to]

### Option C: [Approach 3]
[When to choose this + quick how-to]

## Design Decisions
As you build, decide:
1. [Decision 1]: [Option A] vs [Option B] — consider [tradeoff]
2. [Decision 2]: [Option A] vs [Option B] — consider [tradeoff]
3. [Decision 3]: [Option A] vs [Option B] — consider [tradeoff]

## Quality Checklist
- [ ] Does it handle the happy path?
- [ ] Does it handle errors gracefully?
- [ ] Is it documented enough for someone else to understand?
- [ ] Would you use this yourself?

## Share Your Build
Post your result in the community with:
- Screenshot of the workflow/output
- One thing you'd improve next
- Tag: #workshop-[topic]

## Showcase Examples
[2-3 examples of what great submissions look like]
```

**Video length**: 15-20 min (or async — no video, just the brief)  
**Best for**: Module 5 (Advanced AI), Module 8 (Agent Architecture), Module 11 (Business Process)

## Type 4: Case Study Lesson

**When to use**: Analyzing a real-world implementation to extract lessons and patterns.

```markdown
# Case Study: How [Person/Company] [Achieved Result] with [Method]

## The Situation
- **Who**: [Description of the person/company]
- **Problem**: [What they were struggling with]
- **Scale**: [Numbers that contextualize the challenge]
- **Previous attempts**: [What they tried before]

## The Approach
### Phase 1: [Discovery/Analysis]
[What they did first and why]

### Phase 2: [Build/Implementation]
[The specific tools, workflow, and architecture they used]
[Include architecture diagram description if relevant]

### Phase 3: [Launch/Iteration]
[How they rolled it out and what they learned]

## The Results
| Metric | Before | After | Change |
|--------|--------|-------|--------|
| [Metric 1] | [X] | [Y] | [+/- Z%] |
| [Metric 2] | [X] | [Y] | [+/- Z%] |
| [Metric 3] | [X] | [Y] | [+/- Z%] |

## Key Lessons
1. **[Lesson 1]**: [Explanation + how to apply it]
2. **[Lesson 2]**: [Explanation + how to apply it]
3. **[Lesson 3]**: [Explanation + how to apply it]

## What Would You Do Differently?
[Open-ended question for community discussion]

## Apply This to Your Work
- [ ] Identify one process in your business that matches [pattern from case study]
- [ ] Map out the 3 biggest friction points
- [ ] Pick one to automate this week using [technique from case study]
```

**Video length**: 8-12 min  
**Best for**: Module 6 (Monetize), Module 12 (Teach & Productize), any module as supplementary content
