It is designed for Cowork/Claude Code environments where the Chrome extension is available.## What it does1. Opens the YouTube video URL in Chrome2. Extracts title, description, chapters, and metadata from the page3. Synthesizes a structured `.md` file with: summary bullets, pros, cons, chapter table, and a   bottom-line recommendation## Step-by-step workflow### 1. Check Chrome connectionCall `tabs_context_mcp` with `createIfEmpty: true` to get an available tab. If Chrome is notconnected, inform the user and ask them to reconnect the extension before proceeding. Do not fallback to `WebFetch` for YouTube — it will be blocked.### 2. Navigate to the videoUse `navigate` to load the YouTube URL in the available tab. Wait for the tab title to update tosomething other than "New Tab" before proceeding.### 3. Extract page contentCall `get_page_text` on the tab. The JSON-LD metadata embedded in the page reliably contains:- `name` — the video title- `description` — full description including chapters (timestamped)- `author` — channel name- `uploadDate`- `duration` (ISO 8601 — e.g. `PT14352S` = ~4 hours)- `interactionStatistic` — view count and like countThe chapters are listed in the description as lines like `00:00:00 Chapter Title`. Parse theseinto a clean table.### 4. Generate the summary .mdWrite the output to `/sessions/.../mnt/Sandbox/<slugified-title>-summary.md`.Use the structure below. The goal is a document someone can skim in 2 minutes and know whetherto watch the full video, and which sections to jump to.## Output format```markdown# [Video Title]**By:** [Channel Name] | **Published:** [Date] | **Duration:** [human-readable, e.g. "4h 12m"]**URL:** [original YouTube URL]---## ?? What It Is[2–3 sentence plain-English summary of what the video covers and who it's from]---## ?? Key Topics Covered- [topic 1 — brief description]- [topic 2 — brief description]...---## ? Pros- [pro 1]- [pro 2]...## ? Cons- [con 1]- [con 2]...---## ?? Chapter Breakdown| Timestamp | Chapter ||-----------|---------|| [HH:MM:SS or M:SS] | [Chapter name] |...---## ?? Bottom Line[2–3 sentences: who should watch, what to skip, specific high-value sections to jump to]**Best for:** [audience types]**Skip if:** [when it's not the right resource]```## Pros/cons generation guidelinesBecause YouTube videos don't contain explicit pros/cons lists, infer them from context:**Pros to look for:** depth of coverage, practitioner credibility, free access, practical examples,structured chapters, multi-platform coverage, recency, production quality.**Cons to look for:** length/runtime burden, beginner vs. advanced mismatch, promotional contentor affiliate bias, ethical grey areas (flag clearly), passive format (no exercises), informationthat may date quickly.Be honest — don't manufacture pros/cons that aren't grounded in the actual content signals.## Duration parsingConvert ISO 8601 `PT<N>S` to human-readable:- `PT14352S` › 14352 / 3600 = ~3.98h › "~4h"- `PT2731S` › 2731 / 60 = ~45m › "45m"## Error handling- If `get_page_text` returns no JSON-LD metadata, try reading the visible text sections for title  and description.- If chapters are absent from the description, omit the chapter table and note "No chapters  provided."- If the video is private or unavailable, tell the user clearly and stop.- If Chrome isn't connected, ask the user to reconnect and retry — don't attempt WebFetch  workarounds for YouTube.## Output file namingSlugify the video title: lowercase, spaces › hyphens, strip special chars.Example: "COLD EMAIL COPYWRITING Full Course 2026" › `cold-email-copywriting-full-course-2026-summary.md`Save to the user's Sandbox folder and provide a `computer://` link.
