## sidebar.tsx ‚Äî 19 issues

### Critical

**1. Custom context menus have no keyboard access and no focus trap**
Lines 338‚Äì418 (`RecentChatItem`), 470‚Äì489 (`PinnedChatItem`)
Hand-rolled `menuOpen` popup divs have no `role="menu"` / `role="menuitem"` semantics. Keyboard users cannot reach items; the `window.addEventListener('keydown', close)` listener immediately re-closes the menu on the same keystroke that opened it.
**Fix:** Replace with shadcn `DropdownMenu` (already imported). ‚úÖ Fixed 2026-04-18.

**2. Confirm/cancel buttons have no accessible label + tiny touch targets**
Lines 197‚Äì203
Check and X buttons have no `aria-label`. Touch target is 5 √ó 5 CSS px ‚Äî well below WCAG 2.5.5 minimum.
**Fix:** Add `aria-label="Confirm rename"` / `aria-label="Cancel rename"` and `p-1` padding. ‚úÖ Fixed 2026-04-18.

**3. Hover-only action buttons invisible to keyboard users**
Lines 219‚Äì236
Rename and Delete buttons only render on `hovered === true`. Keyboard users never trigger `onMouseEnter`.
**Fix:** Show on `:focus-within` ‚Äî replace `hovered` state with `group-hover:opacity-100 group-focus-within:opacity-100`. ‚úÖ Fixed 2026-04-18.

### High

**4. `RecentChatItem` / `PinnedChatItem` outer divs not keyboard-navigable**
Lines 329‚Äì421, 447
`<div onClick>` rows have no `role`, no `tabIndex`, no `onKeyDown`. Keyboard users cannot activate chat items.
**Fix:** Add `role="button" tabIndex={0} onKeyDown` handler. ‚úÖ Fixed 2026-04-18.

**5. Mobile hamburger visibility conflict causes SSR flash**
Lines 604‚Äì613
`style={{ display: collapsed ? undefined : 'none' }}` conflicts with `md:hidden` Tailwind class.
**Fix:** Remove inline style; use only Tailwind: `className={cn(!collapsed ? 'hidden' : 'flex md:hidden')}`. ‚úÖ Fixed 2026-04-18.

**6. `window.dispatchEvent(new CustomEvent(...))` coupling**
Lines 807, 887
Raw DOM events bypass React data flow and TypeScript typing. Silently breaks if the listener is renamed.
**Fix:** Lift dialog state to `DashboardShell` and pass as a callback prop, same as `onNewChat`. ‚úÖ Fixed 2026-04-18 ‚Äî added `onShowChatList?` and `onOpenTaskDialog?` props; window events kept as fallback until DashboardShell is wired.

**7. Hardcoded workspace names are dummy data**
Lines 1067‚Äì1079
`['portfolio', 'xeref-claw', 'XerefWhisper-desktop']` are literal strings, non-interactive, shown to all users.
**Fix:** Accept `workspaces: string[]` prop or remove the section until it's wired to real data. ‚úÖ Fixed 2026-04-18 ‚Äî section removed.

**8. Tab strip active-tab label causes layout shift**
Lines 628‚Äì656
`isActive && <span>{tab.label}</span>` widens the button and shifts other tabs on every switch.
**Fix:** Use `grid grid-cols-3` on the strip, or always render label text with `max-w-0 overflow-hidden` transition. ‚úÖ Fixed 2026-04-18 ‚Äî grid layout + sr-only on inactive labels.

### Medium
