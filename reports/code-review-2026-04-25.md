**`app/actions/classroom.ts` lines 119ŌĆō122, 137ŌĆō140 ŌĆö `fetch()` with relative URL in a Server Action**
```ts
await fetch('/api/classroom/embed', { method: 'POST', ... })
```
Calling `fetch('/api/classroom/embed', ...)` with a relative URL inside a Next.js Server Action is undefined behavior on the server ŌĆö Node's `fetch` does not resolve relative URLs. This will silently fail or throw `TypeError: Failed to parse URL` at runtime. The upsert to Pinecone will be skipped when lessons are created or updated, so the search index will be stale.

**Fix:** Either (a) call `upsertLesson()` from `lib/pinecone.ts` directly (it's already imported as `deleteLessonFromPinecone`, just import `upsertLesson` too), or (b) prefix with the absolute base URL from `process.env.NEXT_PUBLIC_SITE_URL`. Option (a) is strongly preferred ŌĆö the HTTP round-trip is unnecessary.

If `searchLessons` throws (e.g., Pinecone rate limit, network error, invalid API key), the route will return an unhandled 500.
